terraform-aws-bedrock-platform Architecture

HCL / Terraform AWS Infrastructure / IaC

Executive Summary

This module deploys a comprehensive Amazon Bedrock AI platform implementing a full Retrieval-Augmented Generation (RAG) architecture. It provisions Knowledge Bases with S3 data sources and configurable chunking, Bedrock Agents with foundation model access and Lambda-backed action groups, content safety Guardrails (content filters, denied topics, PII detection), OpenSearch Serverless vector storage, model invocation logging, and least-privilege IAM roles for each component. It enables organizations to rapidly stand up enterprise-grade generative AI applications with built-in safety controls and observability.

Data Flow

RAG Ingestion Flow

  1. Documents are uploaded to an S3 bucket configured as a data source for the Knowledge Base.
  2. The Knowledge Base ingests documents, splitting them according to the chunking strategy (FIXED_SIZE with configurable max_tokens and overlap_percentage).
  3. Each chunk is embedded using the specified embedding model (e.g., Amazon Titan Embed Text v1).
  4. Embeddings are stored in the OpenSearch Serverless vector index for efficient similarity search.

Query Flow

  1. An application invokes a Bedrock Agent via the agent alias endpoint.
  2. The Agent processes the query using its foundation model (e.g., Claude 3 Sonnet) and configured instructions.
  3. Guardrails filter the input for content safety, denied topics, and PII before processing.
  4. The Agent retrieves relevant context from Knowledge Bases via vector similarity search in OpenSearch.
  5. If action groups are configured, the Agent can invoke Lambda functions for external API calls.
  6. The foundation model generates a grounded response using retrieved context.
  7. Guardrails filter the output before returning it to the user.
  8. Model invocation logging captures the full interaction to S3 for auditing.

Security Controls

  • Least-Privilege IAM — Automatically generated IAM roles per component (knowledge base, agent, logging) with minimal required permissions.
  • Content Filters — Configurable guardrails for HATE, INSULTS, SEXUAL, VIOLENCE, MISCONDUCT with input/output strength levels.
  • Denied Topics — Block specific conversation topics with definitions and examples.
  • PII Detection — Sensitive information filters with BLOCK or ANONYMIZE actions for PII types.
  • OpenSearch Encryption — Encryption policy for the serverless vector collection.
  • OpenSearch Network Policy — Network access controls for the vector store.
  • OpenSearch Access Policy — Data access policy restricting index operations to the knowledge base role.
  • Model Invocation Logging — Complete audit trail of all model interactions stored in S3.
  • Guardrail Versioning — Immutable guardrail versions prevent accidental policy changes.

Production Readiness Checklist

  • Enable model invocation logging (enable_model_invocation_logging = true)
  • Configure guardrails with appropriate content filter strengths for your domain
  • Set up PII filters with ANONYMIZE action for sensitive data handling
  • Define denied topics for off-limits conversation areas
  • Use FIXED_SIZE chunking with appropriate max_tokens for your document types
  • Set overlap_percentage (default 20%) to maintain context across chunks
  • Create agent aliases for stable invocation endpoints (avoid using DRAFT)
  • Review and restrict IAM roles generated by the module
  • Configure OpenSearch Serverless collection with appropriate capacity
  • Set up S3 lifecycle policies on the logging bucket
  • Test guardrail effectiveness with adversarial inputs before production
  • Monitor OpenSearch Serverless OCU consumption for cost management

Deployment

Quick Start

module "bedrock_platform" {
  source = "github.com/kogunlowo123/terraform-aws-bedrock-platform"

  name_prefix       = "my-platform"
  log_s3_bucket_arn = "arn:aws:s3:::my-log-bucket"

  knowledge_bases = {
    docs = {
      name                      = "documentation"
      description               = "Product documentation"
      embedding_model           = "amazon.titan-embed-text-v1"
      s3_data_source_bucket_arn = "arn:aws:s3:::my-docs-bucket"
    }
  }

  agents = {
    assistant = {
      name             = "assistant"
      description      = "Customer assistant"
      foundation_model = "anthropic.claude-3-sonnet-20240229-v1:0"
      instruction      = "You are a helpful assistant."
    }
  }

  opensearch_collection_name = "my-vectors"
}
